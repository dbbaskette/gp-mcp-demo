Get up and running with the Greenplum Model Context Protocol (MCP) Server in minutes.

Prerequisites

  • Greenplum Database: A running Greenplum instance (or PostgreSQL).
  • MCP Client: [Cursor IDE](https://cursor.com) or [Claude Desktop](https://claude.ai/download).
  • Greenplum MCP Server: Download GP MCP Server,  if not using the existing Deployed MCP Server.

  • Rename the downloaded binary to gp-mcp-server 

    Linux or Mac: mv gp-mcp-server-<OS>-<ARCH>  gp-mcp-server
Windows: Right click on the file, click rename and change the name to gp-mcp-server

     

  • External Authorization Server needs to be deployed and configured if oauth2 authentication is used.

2. Configure Cursor in other modes

The above is to configure mcp server in streamable-http mode, to enable stdio mode, add a config to the mcp.json with the similar way (Settings>Tools & MCP >New MCP Server/Add Custom MCP> Add/Edit Config)

In this example, "greenplum-mcp-stdio" is added:



{
  "mcpServers": {
    "tpcds-mcp-http-basicauth": {
      "type": "streamable_http", 
      "url": "http://localhost:8080/mcp",
      "headers": {
        "Authorization": "Basic cmVhZG9ubHlfdXNlcjp2bXdhcmUK"
      }
    },
    "greenplum-mcp-stdio": {
      "type": "stdio",
      "command": "/absolute/path/to/downloaded/gp-mcp-server",
      "env": {
        "GPDB_HOST": "localhost",
        "GPDB_PORT": "5432",
        "GPDB_DATABASE": "postgres",
        "GPDB_USERNAME": "gpadmin",
        "GPDB_PASSWORD": "your_password"
      }
    }
  }
}

3. Verify Installation

Once connected, ask your AI assistant:

"List the databases in my Greenplum cluster."

"What is the status of the cluster segments?"

The server includes built-in tools like list_databases, cluster_status, and explain_query to interact with your Greenplum instance.

Configurations

User can modify the configurations based on their environment and requirements. This can be done pre or post installation

  • Authentication: Configure noauth (default), basic, or oauth2 in ~/.gp_mcp/config.yaml.
  • Logging: Check server logs at ~/.gp_mcp/server.log (or configure via MCP_SERVER_LOG_PATH).
  • Custom Tools: Extend functionality by adding SQL-based tools in ~/.gp_mcp/gpdb-tools.yaml.
