Greenplum DDL Example for Columnar Append-Optimized Table (No Compression)

To get high-quality compression recommendations from an LLM via an MCP server, you need to provide it with "ground truth" data.
Here are the steps to set up the environment, populate it, and then query the LLM for optimization.

Create the Base Table (No Compression)


CREATE TABLE catalog_sales_base (
    cs_sold_date_sk integer,
    cs_item_sk integer,
    cs_order_number integer,
    cs_ext_sales_price numeric(7,2)
)
WITH (appendoptimized=true, orientation=column, compresstype=none)
DISTRIBUTED BY (cs_item_sk);

 Insert Sample Data


INSERT INTO catalog_sales_base SELECT 
    (i % 30) + 1,          -- Repetitive dates (Good for RLE)
    i,                     -- Unique IDs (Bad for compression)
    (i / 10),              -- Semi-repetitive orders
    (random() * 100)::numeric(7,2) 
FROM generate_series(1, 10000) s(i);


Data Analysis and Query Requests

  1. I am using a Greenplum Columnar AO table with no compression. recommend the best compresstype and compresslevel for table catalog_sales_base

Schema and Data Management Examples

In Greenplum, tables and columns may be created using arbitrary, meaningless names or cryptic terminology. To clarify their purpose, comments can be added to both tables and columns.

Schema and Table Creation (Using Abstract Naming)

This section defines the table structures using non-descriptive names. 

Note: There are no foreign key relationships maintained here. LLM should identify the relations.

-- Creating a database with a random ID
CREATE DATABASE IF NOT EXISTS db_v7x_992;

-- Connect to the new DB (in psql)
\c db_v7x_992

-- Create a random schema name
CREATE SCHEMA sc_m4_001;

-- Set the search path so you don't have to reference 'sc_m4_001' constantly
SET search_path TO sc_m4_001, public;

-- 1. Sales Transactions Table
CREATE TABLE tx_data_01 (
    f1 INT,      -- Sale ID
    f2 DATE,     -- Sale Date
    f3 NUMERIC,  -- Amount
    f4 INT,      -- Product Link
    f5 INT,      -- Customer Link
    f6 INT       -- Sales Person Link
) DISTRIBUTED BY (f1);

-- 2. Product Metadata Table
CREATE TABLE ref_a_100 (
    k1 INT, 
    k2 TEXT, 
    k3 TEXT
) DISTRIBUTED BY (k1);

-- 3. Customer Master Table
CREATE TABLE metadata_z_99 (
    c_id INT, 
    c_attr1 TEXT, 
    c_attr2 TEXT, 
    c_attr3 TEXT
) DISTRIBUTED BY (c_id);

-- 4. Sales Staff Table
CREATE TABLE staff_x_007 (
    s_pk INT, 
    s_val1 TEXT, 
    s_val2 TEXT, 
    s_val3 DATE
) DISTRIBUTED BY (s_pk);

Add "Comments" to provide the actual intensions

Use the COMMENT ON command to add comments.

COMMENT ON DATABASE db_v7x_992 IS 'Main Sales Data Warehouse';
COMMENT ON SCHEMA sc_m4_001 IS 'Production Sales Schema - V1';

-- Table Descriptions
COMMENT ON TABLE tx_data_01 IS 'Primary sales transaction log';
COMMENT ON TABLE ref_a_100 IS 'Master reference for product categories';
COMMENT ON TABLE metadata_z_99 IS 'Master customer profile records';
COMMENT ON TABLE staff_x_007 IS 'Employee directory for sales representatives';

-- Column Descriptions (Intentions)
COMMENT ON COLUMN tx_data_01.f1 IS 'Unique Sale ID';
COMMENT ON COLUMN tx_data_01.f4 IS 'Link to Product ID (ref_a_100.k1)';
COMMENT ON COLUMN tx_data_01.f5 IS 'Link to Customer ID (metadata_z_99.c_id)';
COMMENT ON COLUMN tx_data_01.f6 IS 'Link to Sales Rep ID (staff_x_007.s_pk)';

COMMENT ON COLUMN ref_a_100.k2 IS 'Official Product Name';
COMMENT ON COLUMN metadata_z_99.c_attr1 IS 'Customer Full Name';
COMMENT ON COLUMN staff_x_007.s_val1 IS 'Sales Person Name';
